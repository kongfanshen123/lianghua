# 前端数据展示与质量监控方案

## 一、项目概述

为动量策略项目创建一个轻量级前端页面，用于展示数据库中的数据，支持表数据浏览、数据质量监控和策略结果可视化。

## 二、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端API | FastAPI | 轻量高性能，与项目Python环境一致 |
| 前端框架 | 纯HTML+CSS+JS | 无需Node构建链，简单实用 |
| UI样式 | Bootstrap 5 | 快速构建美观界面 |
| 图表库 | ECharts | 轻量级图表，支持柱状图、折线图 |
| 数据库 | SQLite | 项目已有数据库 |

## 三、页面结构

单页面应用，包含三个Tab：

### 1. 数据浏览 Tab
- **标的列表**：symbols表数据，支持搜索和状态筛选
- **每日价格**：daily_prices表数据，按标的筛选，分页展示
- **策略结果**：strategy_results表数据，按日期筛选

### 2. 质量监控 Tab
- **数据质量概览卡片**：覆盖率、异常数、延迟天数等
- **异常明细**：缺失天数、价格跳变、零成交量等详细列表
- **质量趋势图**：每日数据完整性趋势

### 3. 动量排名 Tab
- **动量排名柱状图**：最新交易日各标的动量值排名
- **趋势强度分布**：各强度等级标的数量统计
- **历史排名变化**：近期排名变化趋势

## 四、API接口设计

### 4.1 标的接口
```
GET /api/symbols              # 获取标的列表
GET /api/symbols/{id}         # 获取单个标的详情
```

### 4.2 价格数据接口
```
GET /api/prices               # 获取价格数据（支持分页、标的筛选）
GET /api/prices/symbol/{symbol} # 获取指定标的价格数据
GET /api/prices/count         # 获取价格数据统计
```

### 4.3 策略结果接口
```
GET /api/results              # 获取策略结果（支持日期筛选）
GET /api/results/latest       # 获取最新策略结果
```

### 4.4 数据质量接口
```
GET /api/quality/summary      # 获取数据质量概览
GET /api/quality/details      # 获取数据质量详细检查结果
GET /api/quality/trend        # 获取数据质量趋势
```

## 五、数据质量检查规则

### 5.1 覆盖率检查
- 每个标的最近365天交易日覆盖率 = 有数据天数 / 应交易日数
- 覆盖率低于80%标记为警告，低于60%标记为严重

### 5.2 价格异常检查
- 单日涨跌幅超过30%标记为异常（跳变）
- 价格为0或负数标记为异常

### 5.3 成交量异常检查
- 成交量为0且非停牌标记为异常
- 成交额为0且成交量>0标记为异常

### 5.4 数据延迟检查
- 最新数据日期与当前日期差距超过5个交易日标记为延迟

### 5.5 重复数据检查
- 同一标的同一日期存在多条记录标记为重复

## 六、目录结构

```
frontend/
├── index.html              # 主页面
├── static/
│   ├── css/
│   │   └── style.css       # 自定义样式
│   └── js/
│       ├── main.js         # 主逻辑
│       ├── charts.js       # 图表渲染
│       └── api.js          # API调用封装

backend/
└── api.py                  # FastAPI接口
```

## 七、部署方式

1. 启动API服务：`python3 backend/api.py`
2. 打开浏览器访问：`http://localhost:8000`

## 八、注意事项

1. 前端页面直接通过FastAPI静态文件服务提供
2. API接口无需认证，仅限本地访问
3. 数据量较大时使用分页查询，避免一次性加载过多数据
