# 动量策略后端方案文档

## 1. 项目概述

本项目是一个基于动量交易原理的量化策略后端系统，核心功能包括：
- 每日自动获取常见标的的收盘价数据（支持复权处理）
- 基于20日动量策略计算标的趋势强度并排序（含成交量确认、流动性过滤）
- 将排序结果推送到飞书账号（支持卡片消息、环比变化展示）

### 3.3 核心数据流程

1. **调度触发阶段**：APScheduler每日16:00触发 → 交易日判断 → 进入Pipeline
2. **数据获取阶段**：DataFetcher获取复权收盘价 → 多数据源降级 → 写入数据库
3. **数据校验阶段**：DataValidator校验价格异常/停牌/流动性 → 过滤不合格标的
4. **策略计算阶段**：MomentumStrategy计算20日动量 → 成交量确认 → 排序 → 写入结果表
5. **消息推送阶段**：FeishuNotifier生成卡片消息 → 推送至飞书

## 4. 功能模块设计

### 4.1 数据获取模块 (DataFetcher)

#### 4.1.1 数据源选型

| 数据源 | 适用标的 | 优点 | 缺点 | 优先级 |
|--------|----------|------|------|--------|
| AkShare | A股、港股、基金、ETF | 免费、数据全面、纯Python、支持复权 | 部分接口不稳定 | 主 |
| yfinance | 美股、全球ETF | 免费、海外数据全、自动复权 | 网络延迟高 | 备 |
| Mock | 测试用 | 无外部依赖、可控 | 非真实数据 | 测试 |

**降级策略**：主数据源失败时自动切换至备选数据源，所有数据源均失败时记录错误日志并推送"数据缺失"通知。

**请求限流策略**：
- 单数据源请求间隔 ≥ 1秒
- 失败后指数退避（1s→2s→4s→8s）
- 单个标的最大重试次数 = 3次

#### 4.1.2 复权处理

**复权方式**：采用前复权（Forward Adjustment），确保价格连续性

**复权逻辑**：
```
复权后价格 = 原始价格 × 复权因子
复权因子 = 累计(1 - 分红/除权前价格) × 累计(1 / 拆股比例)
```

**数据源支持**：
- AkShare：直接获取前复权数据（adjust="qfq"）
- yfinance：默认返回复权后数据（auto_adjust=True）

### 4.2 数据校验模块 (DataValidator)

#### 4.2.1 校验规则

| 校验项 | 规则 | 处理方式 |
|--------|------|----------|
| 价格异常检测 | 当日涨跌幅 > ±30% | 标记异常，跳过计算 |
| 停牌过滤 | 成交量 = 0 | 标记停牌，跳过计算 |
| 流动性过滤 | 近20日平均成交量 < 阈值 | 标记流动性不足，跳过计算 |
| 数据完整性 | 缺少收盘价/日期等必填字段 | 标记缺失，跳过计算 |
| 除权除息检测 | 当日发生除权除息 | 标记事件，使用复权数据计算 |

#### 4.2.2 流动性阈值配置

```env
# 流动性过滤配置
MIN_VOLUME_20D=1000000  # 近20日平均成交量阈值（股）
MIN_VOLUME_AMOUNT=5000000  # 近20日平均成交额阈值（元）
```

### 4.3 动量策略模块 (MomentumStrategy)

#### 4.3.1 20日动量计算公式

**标准动量（默认）**：
```
动量值 = (今日收盘价 - 20日前收盘价) / 20日前收盘价 × 100%
```

**指数加权动量（可选）**：
```
动量值 = Σ(权重i × 日收益率i)，其中权重i = 2^(20-i) / Σ(2^k)
```

**公式说明**：
- 正数表示上涨趋势，负数表示下跌趋势
- 绝对值越大，趋势越强
- 指数加权模式赋予近期价格更高权重

#### 4.3.2 成交量确认机制

为过滤量价背离的假突破，增加成交量确认：

| 场景 | 价格趋势 | 成交量趋势 | 确认结果 |
|------|----------|------------|----------|
| 量价齐升 | 上涨 | 放大 | ✅ 有效上涨 |
| 价涨量缩 | 上涨 | 萎缩 | ⚠️ 量价背离，谨慎看多 |
| 量价齐跌 | 下跌 | 放大 | ✅ 有效下跌 |
| 价跌量缩 | 下跌 | 萎缩 | ⚠️ 量价背离，谨慎看空 |

**成交量变化计算**：
```
成交量变化率 = (近5日平均成交量 - 前15日平均成交量) / 前15日平均成交量 × 100%
```

#### 4.3.3 趋势强度等级

| 动量值范围 | 趋势强度 | 说明 |
|------------|----------|------|
| ≥ 15% | 极强上涨 | 强烈看多信号 |
| 5% ~ 15% | 较强上涨 | 看多信号 |
| 0% ~ 5% | 微弱上涨 | 弱看多信号 |
| -5% ~ 0% | 微弱下跌 | 弱看空信号 |
| -15% ~ -5% | 较强下跌 | 看空信号 |
| ≤ -15% | 极强下跌 | 强烈看空信号 |

#### 4.3.4 策略执行流程

```
1. 获取所有启用的标的列表
2. 遍历每个标的：
   a. 查询最近21个交易日的复权收盘价和成交量
   b. 数据校验：价格异常检测、停牌过滤、流动性过滤
   c. 计算20日动量值
   d. 成交量确认：量价一致性验证
   e. 评估趋势强度等级
3. 按动量值从高到低排序
4. 计算排名变化（环比昨日）
5. 保存结果到数据库（幂等性检查）
```

### 4.4 飞书推送模块 (FeishuNotifier)

#### 4.4.1 推送方式

采用**飞书开放平台 App Token**方式推送，需配置：
- APP_ID：飞书应用ID
- APP_SECRET：飞书应用密钥
- CHAT_ID：目标群组/用户ID

**Token缓存策略**：
- tenant_access_token有效期2小时
- 提前10分钟自动刷新
- 缓存至内存，避免重复请求

- 推送失败时自动重试3次

- 全部失败后记录错误日志并发送告警

### 4.5 任务调度模块 (Scheduler)

#### 4.5.1 调度配置

| 任务 | 执行时间 | 说明 |
|------|----------|------|
| 每日Pipeline | 16:00 (收盘后) | 完整流程：fetch→validate→calc→push |
| Token刷新 | 每2小时 | 刷新飞书access_token |

**交易日判断逻辑**：
- 查询交易所日历，判断当日是否为交易日
- A股交易日：周一至周五（节假日除外）
- 美股交易日：周一至周五（节假日除外）
- 非交易日自动跳过，记录日志

## 11. 代码安全性

### 11.1 注意事项

1. **敏感信息保护**：飞书App ID、App Secret等敏感信息通过环境变量配置，禁止硬编码
2. **日志脱敏**：日志中不记录完整敏感信息（如完整的App Secret）
3. **网络安全**：使用HTTPS协议访问外部API，验证SSL证书
4. **数据验证**：对外部数据进行严格校验，防止注入攻击和异常数据
5. **Token安全**：飞书access_token仅缓存于内存，不写入磁盘

### 11.2 错误处理

1. 所有外部API调用均需异常捕获（requests.exceptions）
2. 数据库操作使用事务，确保数据一致性
3. 推送失败时记录详细错误信息，便于排查
4. 关键步骤失败时推送告警，确保及时发现问题
