## 一、验收概述

### 1.1 项目简介

本项目是一个基于动量交易原理的量化策略系统，核心功能包括：每日自动获取标的数据、基于20日动量策略计算趋势强度并排序（含成交量确认、流动性过滤）、提供 Web 可视化管理界面、将结果推送到飞书。

### 1.2 验收方法

| 验收维度 | 验收方式 |
|----------|----------|
| 功能完整性 | 逐模块对照方案文档检查实现情况，通过 Web 界面操作验证 |
| 代码质量 | 全量源码走读（Python ~3,631 行，JS/CSS ~3,339 行） |
| 测试覆盖 | 运行 pytest 测试套件，检查测试用例覆盖面 |
| 运行状态 | 通过 API 接口验证系统实时数据与服务可用性 |
| 数据质量 | 调用质量监控接口，检查数据覆盖率与异常情况 |

## 三、技术负责人验收意见

### 3.1 架构设计评估

#### 架构优点

1. **分层清晰**：数据获取层 → 数据校验层 → 业务逻辑层 → 数据存储层 → 消息推送层 → 任务调度层，各层职责明确，符合单一职责原则。
2. **Pipeline链式触发**：fetch→validate→calc→push 链式执行，前一阶段失败则中断后续流程，避免数据依赖问题。
3. **策略模式应用**：Fetcher、Validator、Strategy、Notifier 均采用抽象基类 + 具体实现的模式，扩展新数据源或策略只需继承基类。
4. **配置集中管理**：通过 `config.py` + `.env` 统一管理配置项，支持环境变量覆盖。
5. **前后端分离**：FastAPI后端 + 静态前端，部署灵活。

#### 架构问题

| 序号 | 问题描述 | 影响 | 优先级 |
|------|----------|------|--------|
| T-01 | `requirements.txt` 缺少 `fastapi` 和 `uvicorn` 依赖，新环境部署后端服务将失败 | 部署阻塞 | 高 |
| T-02 | API中任务触发接口使用 `subprocess.run(["python3", "main.py", "--fetch"])` 调用CLI，但CLI实际使用子命令模式（`main.py fetch`），导致所有Web端任务触发功能失效 | 功能不可用 | 高 |
| T-03 | `database.py` 使用已弃用的 `declarative_base()`（SQLAlchemy 2.0推荐 `sqlalchemy.orm.declarative_base()`），每次启动产生警告 | 技术债务 | 中 |
| T-04 | API服务未配置CORS，跨域访问将被浏览器拦截（前后端分离部署时必现） | 部署隐患 | 中 |
| T-05 | API无任何认证/鉴权机制，所有接口可被未授权访问 | 安全风险 | 中 |
| T-06 | `backfill_data()` 与 `fetch_data()` 存在大段重复代码（数据源选择逻辑、数据写入逻辑），违反DRY原则 | 可维护性差 | 中 |

### 3.2 代码质量评估

#### 代码优点

- **异常处理完善**：Pipeline各阶段均有 try-catch，失败时推送告警并记录日志。
- **幂等性设计**：数据写入前检查已有记录，更新或插入，避免重复数据。
- **日志规范**：关键操作均有日志记录，使用 `get_logger()` 统一管理。
- **数据模型设计合理**：唯一约束、外键关系、字段类型均符合业务需求。
- **前端代码结构化**：API封装、图表逻辑、主逻辑分离为3个JS文件，职责清晰。

### 7.2 数据获取链路正确性与稳定性评估

#### 数据源分布分析

通过数据库直接查询，当前4,712条价格记录的数据来源分布：

| 数据来源 | 记录数 | 占比 | 说明 |
|----------|--------|------|------|
| sina | 3,407 | 72.3% | 新浪财经API获取的真实数据 |
| mock | 1,305 | 27.7% | Mock数据源生成的模拟数据 |

**⚠️ 严重问题：27.7%的数据为Mock模拟数据，且与Sina真实数据混合在同一标的中。**

#### Mock→Sina数据源切换问题

通过逐标的数据溯源，发现**5个标的**存在mock→sina数据源切换导致的极端价格跳变：

| 标的 | 名称 | 切换日期 | Mock最后收盘价 | Sina首日收盘价 | 价格变化幅度 |
|------|------|----------|----------------|----------------|--------------|
| 512880 | 证券ETF国泰 | 2026-07-03 | 103.01 | 1.155 | -98.88% |
| 515030 | 新能源车ETF | 2026-07-03 | 647.26 | 1.882 | -99.71% |
| 512480 | 半导体ETF | 2026-07-03 | 197.22 | 1.331 | -99.33% |
| 512690 | 酒ETF | 2026-07-03 | 78.89 | 0.396 | -99.50% |
| 159928 | 消费ETF | 2026-07-03 | 555.25 | 0.628 | -99.89% |

**根因分析**：

1. **Mock数据价格不真实**：Mock数据源生成的价格（如消费ETF=555.25）与真实市场价格（0.628）差异巨大，Mock数据未基于真实价格范围生成。
2. **数据源切换无衔接**：标的的`data_source`字段从`mock`修改为`sina`后，新获取的真实数据直接追加到历史Mock数据之后，造成价格断崖。
3. **校验器缺陷**：`DataValidator.validate()`仅检查日内价格变化（`close_price` vs `open_price`），不检查日间价格变化（今日`close_price` vs 昨日`close_price`），因此-99%的日间跳变未被拦截。
4. **历史数据未清理**：切换数据源后，Mock历史数据未被清理或标记，直接参与了动量计算。

**512760（芯片ETF国泰）特殊情况**：该标的全程使用sina数据源，但在历史数据中某处存在-49.75%的价格跳变（可能为早期数据获取异常或ETF拆分/合并事件），同样未被校验器拦截。
